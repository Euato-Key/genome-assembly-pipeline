# 前置知识说明

在开始进行基因组组装之前，了解以下基本概念和术语将非常有帮助。

## 1. 测序技术基础

### PacBio HiFi (High Fidelity)
- **特点**: 长读长 (10kb - 25kb) 且高准确度 (>99.9%)。
- **作用**: 是目前基因组组装的**核心数据**。它解决了短读长无法跨越重复序列的问题，同时避免了传统长读长（如 Nanopore）的高错误率。
- **文件格式**: 通常为 `.bam` 或 `.fasta`/`.fastq`。

### Hi-C (Chromatin Conformation Capture)
- **特点**: 捕获染色体内原本在空间上接近的 DNA 片段。
- **作用**: 用于 **Scaffolding (挂载)** 阶段。它可以将组装好的 Contigs（片段）连接成染色体级别的 Scaffolds，并确定它们的顺序和方向。
- **文件格式**: 双端测序数据，`.fastq.gz` (R1, R2)。

### WGS (Whole Genome Sequencing - Illumina)
- **特点**: 短读长 (150bp)，高准确度，成本低。
- **作用**: 在 HiFi 时代，主要用于 **基因组调查 (Survey)**，预估基因组大小、杂合度和重复率。也可以用于最终的 Polishing (纠错)，但对于 HiFi 组装通常不是必须的。

## 2. 组装流程术语

### Contig (重叠群)
- 通过 reads 之间的重叠拼接成的连续序列。中间没有未知碱基 (N)。
- 这是组装的第一步产物。

### Scaffold (支架)
- 将 Contigs 根据配对信息（如 Hi-C）连接起来的序列。
- Contigs 之间可能存在未知序列，用 N 填充。
- 理想情况下，一个 Scaffold 对应一条染色体。

### N50
- 衡量组装连续性的重要指标。
- 定义: 将所有 Contigs 按长度排序，从长到短累加，当累加长度达到总基因组长度的 50% 时，该 Contig 的长度即为 N50。
- 数值越大，说明组装越连续、越完整。

### Busco (Benchmarking Universal Single-Copy Orthologs)
- 衡量组装完整性的指标。
- 通过在组装结果中寻找保守的单拷贝直系同源基因来评估。
- 结果通常表示为: C:98% [S:95%, D:3%], F:1%, M:1%。(C:完整, S:单拷贝, D:重复, F:碎片, M:缺失)。

## 3. 常用软件

- **Fastp**: 速度极快的数据质控和过滤工具。
- **Jellyfish**: K-mer 计数工具，用于基因组调查。
- **GenomeScope**: 基于 K-mer 频率分布图预估基因组特征。
- **Kraken2**: 基于 K-mer 的分类工具，用于检测样本中是否有细菌、病毒等污染。
- **Hifiasm**: 目前最优秀的 HiFi 数据组装软件，支持 Hi-C 辅助定相 (Phasing)。
- **Chromap**: 极速 Hi-C 比对工具。
- **Yahs**: 新一代 Hi-C Scaffolding 工具，速度快且准确。
- **Juicer/Juicebox**: Hi-C 数据处理和可视化工具，用于人工纠错。

## 4. 硬件要求
- **内存 (RAM)**: 基因组组装非常消耗内存。对于哺乳动物基因组 (3Gb)，建议至少 128GB RAM，推荐 256GB 或更多。
- **CPU**: 核心数越多越好，推荐 32 核以上。
- **存储**: 需要大量高速存储 (SSD/NVMe)，建议预留 2TB 以上空间。
